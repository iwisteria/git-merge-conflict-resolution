🔧 Git分岐ブランチ問題の解決方法

対象環境： Mac + VSCode

📋 問題の症状

エラーメッセージ
fatal: Need to specify how to reconcile divergent branches.
hint: You have divergent branches and need to specify how to reconcile them.

これは何？
ローカルブランチとリモートブランチが異なる方向に進んでしまい、Gitが自動的に統合できない状態です。

⚙️
手動解決（コマンドライン）

🛠️ 解決手順（マージ方式）

1 VSCodeでターミナルを開く
  • VSCodeでプロジェクトを開く
  • ターミナル → 新しいターミナル（または ⌘ + Shift + `）
2 マージ設定を行う
git config pull.rebase false

この設定により、今後のプルでマージ方式が使用されます。

3 プルを実行する
git pull origin main

✅ 成功パターン

自動マージ成功の場合：
Merge made by the 'ort' strategy.
example.txt | 2 +-
1 file changed, 1 insertion(+), 1 deletion(-)

→ 完了！ これで問題解決です。

⚠️ コンフリクト発生時の対処

コンフリクトメッセージの例：
CONFLICT (content): Merge conflict in filename.txt
1 競合ファイルを確認
git status
2 競合ファイルを編集

ファイルを開くと以下のような表示があります：

<<<<<<< HEAD
あなたの変更
=======
リモートの変更
>>>>>>> [コミットハッシュ]

不要な行（<<<, ===, >>>）を削除し、残したい内容だけにする

3 変更を保存してコミット
git add .
git commit -m "Merge remote changes"
